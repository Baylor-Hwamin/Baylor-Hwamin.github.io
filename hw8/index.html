N-key calculator experiment result

  • Summary
    • While increasing the number of keys, the total calculating times weredecreased.
    • The difference between the method become increasing while the number of keys are increasing.
    • However, in the case of foot and head, it was getting tired for many numbers of keys. Especially, the 5 keys with head experiment brought little neck pain.
    • In this experiment, I think the the more key can increase the speed for any input method, but it may cause the fatigue of the body. (except the finger or hand based interface)

  • Experiment Method
    • experiments
      • 4 N-key calculators: 1, 2, 3, and 5 key(s) calculator
      • 3 input way for each calculator: Hand, Foot, and Head
      • 3 calculation euations (randomly generated) for each combination.
      • Total 36 times of tests.
    • method
      • Tested with 3 participants (12 tests for each person)
      • Measure the total time for calculating 10 equations.
      • Measure the number of the error. The error was only counted when they 'selected' the wrong key in accident. (Since there were too many moving mistake, I only counted the errors for wrong selection.)
